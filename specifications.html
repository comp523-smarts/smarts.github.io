Specifications

Project Concept

Project Tweet

Trauma Ready: A cross-platform mobile app which helps paramedics navigate the trauma activation criteria of different trauma centers in NC.

User Stories

  • As a User:
    • I want a mobile app so that I can access the information I need on-the-go
    • I want a mobile app that functions on Android and iOS so that I can access the information on either mobile OS
    • I want a display that is visible in low-lighting so I can see the information in dark environments
    • I want to be able to access activation cards for different trauma centers so that I can accurately route patients
    • I want quick information on a particular trauma center so that I don't have to waste important time searching
    • I want a simple, intuitive application so that I can get started without "learning" the application
    • I potentially want trauma center recommendations based on GPS so that I can find the nearest locations without manually searching
    • I want a mobile app that provides me information on the services provided by regional trauma centers so that I can more accurately route patients
    • I want a mobile app that provides me contact information to regional trauma centers so that I can contact them easily
    • I want a mobile app that stores the most important information (trauma activation cards, services, contact information) on the phone, so that my application still works without an internet connection
    • I want a mobile app that still works without an internet connection so that I can use it in areas where there is no internet connection
    • I want a mobile app that provides color coding based on the trauma activation codes so that I can more easily determine the correct trauma activation code for a patient
    • I potentially want a mobile app that provides information on destination protocols so that I can more accurately route patients
    • I potentially want a mobile app that displays distance to closest trauma centers so that I can more quickly route patients
    • I potentially want a mobile app that provides information on how to best transport patients to trauma center so that I can more effectively route patients
    • I potentially want a mobile app that allows me to use search criteria to limit the trauma centers displayed on the application so that I can more effectively find a trauma center and the proper activation codes for that center
    • I potentially want a mobile app that provides a tutorial so that I can quickly learn how to operate the application
    • I potentially want a mobile app that responds to a search query within a reasonable amount of time so that I am not losing time in vital predicaments
    • I potentially want a mobile app that notifies the selected trauma center of the activation code, and related information, so that the patient can be more easily transferred to the trauma center
  • As an Admin:
    • I want to be able to update content on the app without knowing how to code so I can present paramedics with the most relevant information
    • I want to be able to login to a web portal so that only I can update content on the app
    • I want to be able to use a web portal to update content on the app so that it is easier to update information (typing on the computer vs typing on the phone)
    • I want a mobile app that provides the foundation to add more activation criteria, such as for strokes, so that I can expand the usage of the application
    • I want a mobile app that updates every time I make an update to the content so that users always have up-to-date information
    • I would like some form of generic account for each agency so that I can select which trauma centers users (the employees) have access to information on
    • I potentially want a web portal that provides a tutorial/documentation so that I can quickly learn how to update information on the application
    • I potentially want a mobile app that provides the foundation to include more regional trauma centers, and their related information, in North Carolina so that I can expand the usage of the app
    • I potentially want an application that works on desktop/web so that emergency department employees can use the application on their computers

Requirements

Functional

  • Definite:
    • Priority 1
      • Application must provide accurate information on activation code criteria for all trauma centers it records
      • Application must provide accurate information on the services available at different trauma centers
      • Application must provide contact information for each trauma center
    • Priority 2
      • Application may provide information on destination protocols to trauma employees to help determine where to transport trauma patients
      • Application may show closest trauma centers to the user's location, and distance to those trauma centers, via GPS, assuming internet connection is available
      • Application may match trauma experienced by patient with the services available at regional trauma centers to assist the user in deciding where to send the patient
      • Application may allow users to search directly for best-fit activation cards by injury descriptions or trauma center name
  • Perhaps:
    • Application may provide information on how to best transport the patient to the selected trauma center
    • Application may allow users to login/enter a specific code, provided by an Administrator(the user's employer) to gain access to the trauma information of a specific set of regional trauma centers
    • Application may allow administrators to select which regional trauma centers they wish for trauma employees (users) to be allowed access to information on
  • Improbable:
    • Application will notify a selected trauma center automatically of an incoming patient's activation code and the medical justification for that activation code

Non-functional

Usability
  • Perhaps:
    • Application may provide a tutorial for new users on how to operate the application
    • Application's web portal must provide a tutorial/documentation for administrators on how to update information in the application through the web portal
Reliability
  • Definite:
    • Application must continue to function on a basic level (essentially, all "Definite Requirements" still function) without an internet connection
    • Application must function under low-light conditions (contain a daytime and nighttime setting)
    • Application must provide some sort of color coding to match with trauma activation code criteria
Performance
  • Definite:
    • Application must respond to a search query (such as the name of a certain regional trauma center) fairly quickly
Supportability
  • Definite:
    • Application must provide a web portal for administrators to be able to login and update information within the application, without needing any programming knowledge
    Perhaps:
    • Application may update every time an Administrator changes information within the application, in order to keep information up-to-date
    • Application may provide the foundation to add further hospital activation criteria, such as stroke or cardiac activation criteria
    • Application may provide the foundation add more regional trauma centers, and their related information, in North Carolina
Design
  • Definite:
    • Application must function on iOS and Android
    • Application must provide an intuitive layout for users
    • Application connects with a database that stores the information presented on the application, and can be updated through the web portal by an administrator
    Improbable:
    • Application may come in the form of a web/desktop application that can be used by Emergency Department employees on a desktop computer
Interface
  • Perhaps:
    • Application may communicate with a Geolocation API, to assist trauma employee in finding nearby trauma centers
