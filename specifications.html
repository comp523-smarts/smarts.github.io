Specifications

  • Concept
  • Tweet: A cross-platform mobile app which helps paramedics navigate the trauma activation criteria of different trauma centers in NC

User Stories

As a User:
  • I want a cross-platform mobile app so that I can access the information I need on-the-go and from any device
  • I want a high-contrast display that is visible in low-lighting so that I can read the information in dark environments
  • I want to access activation cards for different trauma centers with as few clicks as possible so that I can quickly get the information I need
  • I want a simple user interface so that I can get to my information without the app design getting in the way
  • I want an app that contains activation, service, and contact information for each hospital in one place so I don't have to look it up multiple times
  • I want an app that continues to work without internet so that it's reliable and useful in areas with poor connectivity
  • I potentially want an app that assists me in making my routing decisions by providing recommendations based on GPS
  • I potentially want to view activation criteria as listed on the app so that I can get the information online as well as on mobile
  • I potentially want to search for activation cards by trauma criteria so that I can find relevant trauma centers without having to guess and check
  • I potentially want an app that provides information on routing and traffic so that I can get directions more efficiently
  • I potentially want a mobile app that notifies the selected trauma center of the activation code and related information so that the patient can relieve faster treatment upon arriving
As an Admin:
  • I want to update activation criteria on the app without having a technical background so that I can present paramedics with the latest information
  • I want the mobile app to refresh quickly when I make administrator updates so all paramedics are immediately receiving the same information
  • I want to use a login-based web portal to update my criteria so I can input information easily via keyboard
  • I potentially want a quick tutorial on how to use the web portal so I understand how my actions affect the app better

Requirements

Functional

  • Definite:
    • Priority 1
      • Application must provide accurate information on activation code criteria for all trauma centers it records
      • Application must provide accurate information on the services available at different trauma centers
      • Application must provide contact information for each trauma center
    • Priority 2
      • Application must provide information on destination protocols to trauma employees to help determine where to transport trauma patients
      • Application must show closest trauma centers to the user's location, and distance to those trauma centers, via GPS, assuming internet connection is available
      • Application must match trauma experienced by patient with the services available at regional trauma centers to assist the user in deciding where to send the patient
      • Application must allow users to search directly for best-fit activation cards by injury descriptions or trauma center name
  • Perhaps:
    • Application may provide information on how to best transport the patient to the selected trauma center
    • Application may allow users to login/enter a specific code, provided by an Administrator(the user's employer) to gain access to the trauma information of a specific set of regional trauma centers
    • Application may allow administrators to select which regional trauma centers they wish for trauma employees (users) to be allowed access to information on
  • Improbable:
    • Application will notify a selected trauma center automatically of an incoming patient's activation code and the medical justification for that activation code

Non-functional

Usability
  • Perhaps:
    • Application may provide a tutorial for new users on how to operate the application
    • Application's web portal must provide a tutorial/documentation for administrators on how to update information in the application through the web portal
Reliability
  • Definite:
    • Application must continue to function on a basic level (essentially, all "Definite Requirements" still function) without an internet connection
    • Application must function under low-light conditions (contain a daytime and nighttime setting)
    • Application must provide some sort of color coding to match with trauma activation code criteria
Performance
  • Definite:
    • Application must respond to a search query (such as the name of a certain regional trauma center) fairly quickly
Supportability
  • Definite:
    • Application must provide a web portal for administrators to be able to login and update information within the application, without needing any programming knowledge
    Perhaps:
    • Application may update every time an Administrator changes information within the application, in order to keep information up-to-date
    • Application may provide the foundation to add further hospital activation criteria, such as stroke or cardiac activation criteria
    • Application may provide the foundation add more regional trauma centers, and their related information, in North Carolina
Design
  • Definite:
    • Application must function on iOS and Android
    • Application must provide an intuitive layout for users
    • Application connects with a database that stores the information presented on the application, and can be updated through the web portal by an administrator
    Improbable:
    • Application may come in the form of a web/desktop application that can be used by Emergency Department employees on a desktop computer
Interface
  • Definite:
    • Application must communicate with a Geolocation API, to assist trauma employee in finding nearby trauma centers
